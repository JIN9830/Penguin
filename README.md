<summary>🧩 추상 클래스를 활용한 코딩 블록 설계 👈[접기/펼치기]</summary>
  
  ### 🧩 추상 클래스의 다형성을 활용한 코딩 블록

  게임에서 유저가 자주 사용하는 코딩 블록은 종류가 다양하며 코드 내부적으로도 자주 호출됩니다.

  이러한 이유로 코딩 블록의 기본 동작을 추상 클래스로 정의하고, 하위 클래스에서 이를 구체화하는     방식으로 설계했습니다.

  이를 통해 여러 하위 클래스의 객체를 하나의 추상 클래스 타입으로 다룰 수 있습니다. 결과적으로      다양한 종류의 코딩 블록을 하나의 타입으로 호출할 수 있어 코드의 재사용성이 향상됩니다.
  
  ```
  코딩 블록 추상 클래스 구조
  
  1. CodingBlock.cs [Abstract]
  
  2. Forward.cs [Class : Abstract]
     
  4. TurnRight.cs [Class : Abstract]
  
  5. TurnLeft.cs [Class : Abstract]
  
  6. Funtion.cs [Class : Abstract]
  
  7. Loop.cs [Class : Abstract]
  ```

  ***추상 클래스와 블록들의 관계도 그림***

  ***추상 클래스 내부 코드 사진 (추상 클래스내부의 가상 메소드 사진)***
  
  추상 클래스를 통해서 모든 코딩블록들이 가져야 할 공통적인 메서드를 구현하여 상속 받는 모든 하위
  클래스들은 이 메서드를 사용할 수 있습니다.

  ***추상 클래스 내부 코드 사진 (추상 클래스의 가상 메서드와 하위 클래스의 일반 메서드)***
  
  이를 통해서 하위 클래스에서 필요한 부분만 구현하고, 공통된 부분은 그대로 사용하거나 확장할 수 
  있어 시스템의 확장성을 높입니다.
  
  **일관성 있는 설계**
 
  추상 메서드를 통해서 하위 클래스가 반드시 구현해야 할 동작을 강제할 수 있습니다.
  
  이를 통해서 클래스 간의 일관성 있는 인터페이스를 유지할 수 있습니다.

  **다형성 활용**

  추상 클래스는 다형성을 지원하여, 상속받은 여러 하위 클래스를 하나의 추상 클래스 타입으로 묶어서   
  다룰 수 있게 합니다. 이는 유연하고 확장 가능한 설계를 가능하게 합니다.

  **구조적인 설계 지원**

  템플릿 메서드 패턴과 같이, 전체적인 알고리즘의 구조는 추상 클래스에 정의하고, 세부 구현은 하위    클래스의 맡기는 방식으로 설계하였습니다.


**추상 클래스는 코드 재사용성을 높이고, 구현을 강제하며, 시스템 확장성을 향상합니다.**
    
추상 클래스는 다형성을 지원하여, 상속 받은 여러 하위 클래스를 하나의 추상 클래스 타입으로 묶어 다룰 수 있게 합니다. 
    
이는 유연하고 확장 가능한 설계를 가능하게 합니다.

## 💡 트러블 슈팅 (Troubleshooting)
지원자의 문제 해결 능력을 보여주는 가장 중요한 파트입니다. 단순히 '만들었다'에서 끝나는 게 아니라, '어려움이 있었지만 이렇게 해결했다'를 보여주는 것이 실무자들에게 깊은 인상을 남깁니다.

문제 상황: 개발 중 발생했던 가장 기억에 남는 문제점 (예: 모바일에서 프레임 드랍이 심각하게 발생, 특정 기기에서 UI가 깨지는 현상 등)

원인 분석: 문제를 해결하기 위해 어떤 가설을 세우고, 어떤 방법(프로파일러, 로그 분석 등)으로 원인을 찾아냈는지 설명

해결 과정: 찾아낸 원인을 어떤 기술적/논리적 방법으로 해결했는지 구체적으로 서술

결과 및 배운 점: 문제 해결 후 어떤 결과(성능 개선 수치 등)를 얻었으며, 이 경험을 통해 무엇을 배우게 되었는지 작성






## 📋 프로젝트 회고 (Post-mortem)
프로젝트를 통해 무엇을 배웠고, 어떤 점이 아쉬웠는지 솔직하게 작성하며 성장 가능성을 보여주는 부분입니다.

잘한 점: 이번 프로젝트에서 스스로 칭찬하고 싶은 부분 (예: 체계적인 Git 관리, 새로운 기술을 성공적으로 학습하고 적용한 점 등)

아쉬운 점: 다시 만든다면 개선하고 싶은 부분 (예: 코드 구조를 더 객체지향적으로 설계, 기획 단계에 시간을 더 투자하고 싶은 점 등)

배운 점: 이 프로젝트를 통해 얻게 된 기술적, 협업적 교훈
